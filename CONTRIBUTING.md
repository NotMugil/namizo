## Ways to Contribute

There are several ways you can contribute to this project:

1. Report issues: If you find a bug or want to request a feature, open an issue on GitHub.
2. Submit fixes: Fork the repository, make your changes, and open a pull request.
3. Improve documentation: Clarify setup steps, commands, or architecture notes for other contributors.

## How to Submit a Pull Request

To submit a pull request (PR):

1. Fork this repository.
2. Clone your fork locally:

   ```bash
   git clone https://github.com/NotMugil/namizo.git
   ```

3. Create a branch for your changes:

   ```bash
   git checkout -b feat/your-change
   ```

4. Make your changes and test them thoroughly.
5. Commit with a clear, descriptive message. Example:

   ```text
   feat: add desktop provider parser scaffolding
   ```

6. Push your branch and open a PR against `main`.
7. Include context in your PR description:
   - what changed
   - why it changed
   - how it was tested
   - screenshots/videos for UI changes (if applicable)

## Coding Conventions

- Follow existing code style and project structure.
- Keep commits and pull requests focused on a single purpose.
- Keep Flutter (`mobile/`) and Tauri (`desktop/`) changes independent unless the PR clearly requires both.
- Add or update tests when introducing behavior changes.
